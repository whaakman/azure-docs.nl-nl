# <a name="contributing-to-microsoft-azure-documentation"></a>Een bijdrage levert aan Microsoft Azure-documentatie

> Tussen 5 en 9 maart 2019 wordt de opslagplaats MicrosoftDocs/azure-docs gemigreerd van zijn huidige Markdown-engine naar de markdown-engine [Markdig](https://github.com/lunet-io/markdig). Gedurende deze tijd schakelen we de mogelijkheid uit om samenvoegingen van openbare pull-aanvragen te onderbreken, zodat we de migratie kunnen voltooien.

Hartelijk dank voor uw bijdragen aan de Microsoft Azure-documentatie.

Deze handleiding bevat informatie over sommige algemene onderwerpen over bijdrage en verwijst uit naar onze [gids voor inzenders](https://docs.microsoft.com/contribute) voor meer gedetailleerde uitleg, indien van toepassing.

## <a name="code-of-conduct"></a>Gedragscode

Op dit project is de [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/) (Microsoft Open Source-gedragscode) van toepassing.
Zie voor meer informatie de [Code van Conduct FAQ](https://opensource.microsoft.com/codeofconduct/faq/), of neem contact op met [ opencode@microsoft.com ](mailto:opencode@microsoft.com) aanvullende vragen of opmerkingen hebt.

## <a name="how-can-i-contribute"></a>Hoe kan ik bijdragen?

Er zijn verschillende manieren om bij te dragen aan de documentatie, Controleer de secties hieronder in om te achterhalen welke is geschikt voor u.

### <a name="reporting-bugs-and-suggesting-enhancements"></a>Rapportage van Bugs en verbeteringen voorstellen

Gebruik het hulpprogramma Feedback onder aan elk artikel om in te dienen bugs en suggesties.

![Feedback-hulpprogramma](media/feedback-tool.png)

### <a name="editing-in-github"></a>Bewerken in GitHub

Volg de instructies voor [snel wijzigingen in bestaande documenten](https://docs.microsoft.com/contribute/#quick-edits-to-existing-documents) in onze gids voor inzenders.

### <a name="pull-request"></a>Pull-aanvraag

Bekijk de gids voor [Pull-aanvragen](https://docs.microsoft.com/contribute/how-to-write-workflows-major#pull-request-processing) in onze gids voor inzenders.
